# Pilot Results — Sales Artifact Preview

This directory contains a **standalone HTML preview** of the Pilot Results
PDF that the sales motion hands to prospects after a 90-day pilot.

## Files

- `pilot-results.html` — A self-contained HTML file with **sample data**
  baked in. Open it directly in any browser to see exactly what the PDF
  looks like when generated from real `/pilot-results` data.

## How the real PDF is generated

The PDF is generated by `exportPilotResultsPDF()` in
`src/lib/export/pdf.ts`. It opens a new tab with the same HTML template
populated with real snapshot data, and calls `window.print()` so the user
can save as PDF via their browser's print dialog.

## Visual layout

1. **Cover** — WPOS logo + wordmark, serif title, organisation name,
   italic tagline ("Workforce performance, explainable."), pilot window
   pill, indigo gradient rule.
2. **Summary stats** — 4 tiles: KPIs Tracked, Improved, Declined, Avg Δ.
3. **Callouts** — Top improvement (green) and Needs attention (red),
   auto-derived from the data.
4. **Per-KPI Performance** — table with: KPI/Employee, Baseline,
   Current, Trajectory bar (grey baseline vs green/red current),
   Δ%, Window.
5. **Methodology box** — explains what the numbers mean and how
   improvement is calculated.
6. **Footer** — WPOS brand mark, "Confidential", generation date.

## Previewing locally

```bash
open preview/pilot-results.html      # macOS
xdg-open preview/pilot-results.html  # Linux
start preview/pilot-results.html     # Windows
```

To save as PDF from the preview: print dialog → "Save as PDF" → enable
**Background graphics** (Chrome hides gradients/colors without it).
